# 🧰 Dev Team Skills — مخزن شخصی اسکیل‌های حرفه‌ای

یک «تیم مجازی حرفه‌ای» برای سپردن پروژه به ایجنت‌های AI با خیال راحت:
تمیزی کد ✅ ترتیب کارها و پلن‌ها ✅ ترسیمات شاخه‌ای ✅ حافظه‌ی حرفه‌ای ✅ پروتکل دیباگ ✅

> الهام‌گرفته از الگوهای خوب پروژه‌ی **ruflo** (memory ،checkpoints ،متد سیستماتیک) — اما سبک و فایل‌محور، بدون پیچیدگی‌های AgentDB و vector DB.

## 🧠 حالت SMART (توصیه‌شده)

فقط SMART را فعال کن — بقیه خودکار است:
1. پروژه را **اسکن** می‌کند (فایل‌ها، پلن، حافظه)
2. **فاز** (P0 ایده → P6 نگهداری) و **سطح** (S/M/L) را تشخیص می‌دهد
3. اسکیل‌های مناسبِ همان لحظه را **فعال** و بقیه را **صف** می‌کند
4. «بسته‌ی هوشیاری» (حافظه + گیت) را **همیشه روشن** نگه می‌دارد
5. قطع شدی؟ فراخوانی بعدی SMART = **ادامه از همان‌جا**، نه از صفر
6. گزارش هر اجرا: **درختی 🌳 + جدول ساده 📋**

## 🚀 نحوه‌ی استفاده

### با claude.ai (Skills)
هر پوشه‌ی داخل `skills/` را جداگانه ZIP کن و در Settings → Capabilities → Skills آپلود کن،
یا کل مخزن را به‌عنوان Project Knowledge بده.

### با هر ایجنت دیگر (Genspark ،Cursor ،Cline ،...)
1. این ریپو را کنار پروژه‌ات قرار بده (یا لینک بده)
2. محتوای `AGENT_PROMPT.md` را در ابتدای کار به ایجنت بده
3. ایجنت با `00-team-orchestrator` شروع می‌کند و بقیه خودکار پیش می‌رود

## 🔄 چرخه‌ی کاری که تضمین می‌شود

```
شروع جلسه → خواندن حافظه → پلن → معماری → کدِ تمیز
   ↑              گیر کرد؟ → پروتکل دیباگ (نه حدس!)          
   │              تغییر داد؟ → کامیت فوری
   └── پایان جلسه ← ثبت حافظه ← تست واقعی ← checkpoint
```

## ⛔ ۷ قانون آهنین (خلاصه)

1. کد بدون پلن ❌
2. جلسه بدون خواندن حافظه ❌
3. جلسه بدون به‌روزرسانی حافظه ❌
4. تلاش چهارمِ حدسی روی یک باگ ❌ (→ پروتکل دیباگ)
5. تغییرِ بدون کامیت ❌
6. «تمام شد» بدون تست واقعی ❌
7. انحراف از فاز فعلی ❌ (→ پارکینگ ایده‌ها)
