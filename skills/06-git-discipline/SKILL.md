# Git Discipline — نظم نسخه‌بندی

هدف: در هر لحظه بتوان با خیال راحت به آخرین حالت سالم برگشت. git «شبکه‌ی ایمنی» پروژه است.

## ⚡ قوانین پایه

1. **هر تغییر معنادار = یک کامیت. بلافاصله.** هیچ تغییری شب را uncommitted نمی‌گذراند.
2. **کامیت اتمیک:** یک کامیت = یک منظور. «رفع باگ + افزودن فیچر + فرمت» در یک کامیت ممنوع.
3. **قبل از کامیت:** چک‌لیست `04-clean-code` + اجرای formatter/تست‌های سریع.
4. **هرگز روی کد خراب کامیت «wip» انباشته نکن** — اگر وسط کاری، حداقل پیام دقیق بنویس: `wip(auth): فرم لاگین — اعتبارسنجی مانده`

## 📝 قالب پیام کامیت (Conventional Commits)

```
<type>(<scope>): <خلاصه‌ی امری و دقیق>

[بدنه‌ی اختیاری: چرا این تغییر؟]
```

| type | کاربرد |
|---|---|
| feat | قابلیت جدید |
| fix | رفع باگ |
| refactor | بهبود ساختار بدون تغییر رفتار |
| test | افزودن/اصلاح تست |
| docs | مستندات (شامل PLAN, ARCHITECTURE, memory) |
| chore | ابزار، کانفیگ، وابستگی‌ها |

مثال‌ها:
```
feat(auth): افزودن endpoint لاگین با JWT
fix(items): رفع خطای null هنگام لیست خالی — ر.ک lessons.md#L-004
docs(memory): checkpoint فاز ۱ + به‌روزرسانی state
```

## 🏁 Checkpoint = تگ + ثبت در حافظه

پایان هر فازِ تست‌شده:

```bash
git add -A
git commit -m "feat(phase-1): تکمیل MVP — همه تست‌ها سبز"
git tag -a phase-1 -m "Checkpoint: Phase 1 complete"
```

سپس در `checkpoints.md` ثبت کن (هش کامیت + دستور برگشت امن).

## 🔀 شاخه‌ها (ساده نگه دار)

- پروژه‌ی تک‌نفره: `main` + شاخه‌ی کاری (مثل `dev` یا `genspark_ai_developer`)
- آزمایش پرریسک → شاخه‌ی موقت `experiment/<نام>`؛ جواب داد merge، نداد حذف — main همیشه سالم می‌ماند
- قبل از PR: sync با remote، حل конфلیکت (اولویت با remote)، squash کامیت‌های ریز به یک کامیت جامع

## ⛔ ممنوعیات

- `git add .` کورکورانه بدون `git status` — فایل ناخواسته (راز، node_modules، فایل موقت) کامیت نشود
- کامیت راز/کلید API — همیشه `.gitignore` را اول پروژه بساز
- force push روی main
- rewrite تاریخچه‌ی publish‌شده (به‌جز squash قبل از PR روی شاخه‌ی خودت)

## 🧰 .gitignore حداقلی هر پروژه

```
node_modules/
.env
*.log
dist/
__pycache__/
.DS_Store
```
(`.project-memory/` عمداً ignore **نمی‌شود** — بخشی از پروژه است.)
