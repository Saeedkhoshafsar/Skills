# Debug Protocol — وقتی گیر کردی، بی‌راه نزن

ذهنیت: **دیباگر متدیک، نه قمارباز.** هر تغییرِ حدسی بدون شواهد = بدهی جدید.

## 🛑 قانون توقف

بعد از **۳ تلاش ناموفق** روی یک مشکل، توقف اجباری و ورود به این پروتکل.
تلاش چهارمِ حدسی ممنوع است.

## 🧭 مرحله ۰ — اول به این‌جاها سر بزن (به همین ترتیب)

1. **`lessons.md`** — این مشکل یا مشابهش قبلاً حل شده؟
2. **`decisions.md`** — آیا با یک تصمیم قبلی می‌جنگی؟ (شاید مشکل، طراحی است نه باگ)
3. **`ARCHITECTURE.md`** — آیا داده واقعاً مسیری را می‌رود که فکر می‌کنی؟
4. **`git log` + آخرین checkpoint** — آخرین حالت سالم کی بود؟ از آن موقع چه تغییر کرده؟
5. **پیام خطا را کامل بخوان** — سطرِ دقیق، stack trace کامل، نه فقط خط اول.

## 🔬 مرحله ۱ — بازتولید و ایزوله‌سازی

- مشکل را **قابل‌تکرار** کن: دقیق‌ترین ورودی/مسیری که خطا می‌دهد
- کوچک‌ترین نمونه‌ی بازتولید بساز (minimal repro)
- **نصف‌کردن (bisect):** نیمی از مسیر را چک کن — مشکل قبل از این نقطه است یا بعدش؟ تکرار کن تا محاصره شود
- لاگ هدفمند بگذار: مقدار واقعی متغیرها در مرزهای مشکوک (ورودی تابع، خروجی DB، پاسخ API)

## 🧪 مرحله ۲ — فرضیه و آزمایش

هر تلاش باید این قالب را داشته باشد (در ذهن یا یادداشت):

```
فرضیه: خطا به‌خاطر X است، چون شاهدِ Y را دیدم.
آزمایش: اگر Z را تغییر دهم، باید W اتفاق بیفتد.
نتیجه: [تأیید / رد] → فرضیه‌ی بعدی بر اساس یافته‌ها
```

- **هر بار فقط یک چیز تغییر کن.** دو تغییر هم‌زمان = نمی‌فهمی کدام اثر کرد
- تغییری که کمکی نکرد را **برگردان** (revert) — انباشت تغییرات حدسی ممنوع

## 🌐 مرحله ۳ — منابع بیرونی

- پیام خطای دقیق را جست‌وجو کن (با نام framework و نسخه)
- مستندات رسمی همان نسخه‌ای که استفاده می‌کنی — نه نسخه‌ی قدیمی
- changelog/breaking changes اگر تازگی نسخه‌ای آپدیت شده

## 🆘 مرحله ۴ — تشدید (Escalation)

اگر بعد از پروتکل هنوز حل نشد، **صادقانه به کاربر گزارش بده**:

```
🔴 گیر کرده‌ام — گزارش وضعیت:
- مشکل: [یک جمله]
- چه چیزهایی امتحان شد: [لیست + نتیجه‌ی هرکدام]
- محتمل‌ترین علت فعلی: [فرضیه]
- گزینه‌ها: (الف) [راه‌حل جایگزین] (ب) [دور زدن موقت] (ج) [نیاز به تصمیم/دسترسی از شما]
```

هرگز وانمود نکن حل شده. هرگز بی‌سروصدا مسیر پروژه را عوض نکن.

## 📝 مرحله ۵ — ثبت درس (اجباری)

هر مشکل حل‌شده (یا بن‌بست) → ورودی در `lessons.md`:
علامت، علت ریشه‌ای، راه‌حل، درس. این همان چیزی است که دفعه‌ی بعد در «مرحله ۰» نجاتت می‌دهد.
