# Project Memory — حافظه‌ی حرفه‌ای پروژه

هدف: ایجنت مثل یک عضو تیم که دفترچه‌ی مهندسی دارد، **همه‌چیز را ثبت کند**:
چه کرد، چرا کرد، چه یاد گرفت، کجا مانده. نتیجه: آگاهی منسجم، بدون تکرار اشتباه، بدون گم‌شدن.

## 📁 ساختار حافظه

در ریشه‌ی هر پروژه، پوشه‌ی `.project-memory/` با این فایل‌ها:

```
.project-memory/
├── state.md           ← وضعیت لحظه‌ای: کجاییم، کار بعدی چیست
├── worklog.md         ← ثبت هر کاری که ایجنت انجام می‌دهد (append-only)
├── decisions.md       ← لاگ تصمیم‌های معماری با «چرا»
├── checkpoints.md     ← نقاط عطف تأییدشده (فاز تمام و تست شد ✅)
├── lessons.md         ← درس‌ها: باگ‌های حل‌شده، بن‌بست‌ها، راه‌حل‌ها
└── ideas-parking.md   ← پارکینگ ایده‌ها (جلوگیری از انحراف وسط کار)
```

## ⏱️ چه زمانی چه چیزی ثبت شود

| رویداد | فایل | الزام |
|---|---|---|
| شروع جلسه | خواندن `state.md` و اعلام «آخرین بار اینجا بودیم» | 🔴 اجباری |
| هر تسک انجام‌شده | یک خط در `worklog.md` | 🔴 اجباری |
| هر تصمیم معماری/فنی | ورودی در `decisions.md` | 🔴 اجباری |
| اتمام و تست هر فاز | ورودی در `checkpoints.md` | 🔴 اجباری |
| حل هر باگ سخت / بن‌بست | ورودی در `lessons.md` | 🔴 اجباری |
| ایده‌ی جدید وسط کار | `ideas-parking.md` — و ادامه‌ی کار فعلی | 🔴 اجباری |
| پایان جلسه | به‌روزرسانی کامل `state.md` | 🔴 اجباری |

## 📄 قالب‌ها

### state.md
```markdown
# وضعیت پروژه
- **فاز فعلی:** Phase 2 — احراز هویت
- **تمام‌شده:** Phase 1 (اسکلت پروژه + DB) ✅
- **در حال انجام:** فرم لاگین — اعتبارسنجی سمت سرور مانده
- **کار بعدی مشخص:** اتصال JWT به middleware
- **موانع باز:** خطای CORS در محیط dev (جزئیات در lessons.md#3)
- **آخرین به‌روزرسانی:** 2026-07-06 — جلسه ۴
```

### worklog.md (append-only — هرگز خط قبلی پاک نشود)
```markdown
| تاریخ | جلسه | کار انجام‌شده | نتیجه |
|---|---|---|---|
| 2026-07-06 | 4 | ساخت endpoint /api/login | ✅ تست شد |
| 2026-07-06 | 4 | تلاش برای رفع CORS | ⏸️ ناتمام — ثبت در lessons |
```

### decisions.md

- **زمینه:** پروژه تک‌کاربره است، دیپلوی ساده اولویت دارد

- **چرا:** صفر پیکربندی؛ مهاجرت بعدی آسان است
- **بدیل‌های ردشده:** Postgres (سنگین برای MVP)
```

- **تست:** همه‌ی تست‌های واحد سبز — اجرای دستی OK
- **برگشت امن به این نقطه:** `git checkout a1b2c3d`
```

### lessons.md
```markdown
## L-003: خطای CORS در dev
- **علامت:** درخواست‌های fetch از فرانت بلاک می‌شدند
- **علت ریشه‌ای:** پورت dev فرانت در whitelist نبود
- **راه‌حل:** افزودن origin به کانفیگ cors در server.js
- **درس:** هر سرویس جدید = اول کانفیگ CORS چک شود
```

## ⛔ قوانین

1. حافظه **قابل‌اعتمادتر از ذهن ایجنت** است — اگر تضاد بود، حافظه برنده است.
2. `worklog.md` فقط اضافه می‌شود، هرگز ویرایش/حذف نمی‌شود.
3. ورودی‌ها **کوتاه و دقیق** — این حافظه است، نه رمان.
4. قبل از دیباگ هر مشکل، اول `lessons.md` جست‌وجو شود — شاید قبلاً حل شده.
5. پوشه‌ی `.project-memory/` همیشه در git کامیت می‌شود.
