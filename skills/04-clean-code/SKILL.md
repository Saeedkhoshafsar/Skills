# Clean Code — استاندارد تمیزی کد

ذهنیت: **سینیور دولوپری که می‌داند کد بیشتر خوانده می‌شود تا نوشته.**

## 📛 نام‌گذاری

- نام‌ها **قصد را بیان کنند**: `remainingRetries` نه `n`؛ `isEmailValid` نه `flag`
- توابع = فعل (`sendInvoice`)، متغیرها = اسم (`invoiceTotal`)، بولین‌ها = سوال (`hasAccess`, `isReady`)
- یک مفهوم = یک واژه در کل پروژه (نه `fetch/get/retrieve` قاطی)
- نام مبهم = نشانه‌ی طراحی مبهم؛ اول طراحی را شفاف کن

## 🔧 توابع

- **یک تابع = یک کار.** اگر «و» در توضیحش هست، دو تابع است.
- کوتاه (هدف: زیر ~۳۰ خط)، حداکثر ۳ پارامتر (بیشتر → آبجکت options)
- بدون side effect پنهان — اسم `getUser` نباید چیزی بنویسد
- Early return به‌جای if تو در تو:

```js
// ❌
function process(user) {
  if (user) {
    if (user.isActive) { /* منطق */ }
  }
}
// ✅
function process(user) {
  if (!user?.isActive) return;
  /* منطق */
}
```

## 🚨 مدیریت خطا

- هیچ `catch` خالی — حداقل لاگ با زمینه (چه عملیاتی، با چه ورودی‌ای)
- خطاها در **مرز سیستم** (ورودی کاربر، API خارجی، فایل/DB) هندل شوند
- پیام خطای مفید: «اتصال به DB شکست خورد در getUser(id=42): timeout» نه «error!»
- Fail fast: ورودی نامعتبر را همان اول رد کن، نگذار عمیق در سیستم بترکد

## 💬 کامنت‌ها

- کامنت «چرا» را می‌گوید، کد «چه» را — کامنتی که کد را تکرار کند حذف شود
- کد کامنت‌شده (dead code) ممنوع — git تاریخچه را دارد، پاکش کن
- برای هر workaround یا محدودیت: کامنت `// NOTE:` یا `// HACK:` با دلیل و لینک به `lessons.md`

## 🧹 اصول ساختاری

- **DRY منطقی:** منطق تکراری را استخراج کن، ولی از انتزاع زودرس بپرهیز (قانون ۳ بار: بار سوم تکرار = وقت استخراج)
- **KISS/YAGNI:** ساده‌ترین راهی که کار می‌کند؛ چیزی که «شاید بعداً لازم شود» نساز
- ثابت‌های جادویی ممنوع: `MAX_RETRIES = 3` نه `3` وسط کد
- کانفیگ (پورت‌ها، کلیدها، URLها) در env/کانفیگ، هرگز هاردکد
- فرمت یکدست: اگر formatter هست (prettier, black, gofmt) قبل از هر کامیت اجرا شود

## ✅ چک‌لیست خودبازبینی (قبل از هر کامیت)

- [ ] نام‌ها بدون نیاز به توضیح قابل‌فهم‌اند؟
- [ ] هر تابع فقط یک کار می‌کند؟
- [ ] مسیرهای خطا هندل شده‌اند (نه فقط happy path)؟
- [ ] کد مرده، console.log دیباگ، یا TODO بی‌صاحب باقی نمانده؟
- [ ] راز (کلید API، پسورد) در کد نیست؟
- [ ] با `ARCHITECTURE.md` سازگار است (لایه‌ی درست، وابستگی درست)؟
- [ ] اگر ۶ ماه دیگر برگردم، می‌فهممش؟
