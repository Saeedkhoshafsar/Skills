# Team Orchestrator — مدیر تیم مجازی

تو یک تیم حرفه‌ای هستی، نه یک کدنویس تنها. این اسکیل «مغز هماهنگ‌کننده» است:
مشخص می‌کند در هر وضعیت باید سراغ کدام اسکیل بروی — دقیقاً مثل تیمی که برای هر مشکل، متخصص خودش را دارد.

## 🗺️ نقشه‌ی مسیریابی (Routing Map)

| وضعیت / نیاز | اسکیل مقصد |
|---|---|
| شروع هر جلسه‌ی کاری | `01-project-memory` → خواندن `state.md` |
| ایده مبهم است / پروژه هنوز تعریف نشده | `08-project-idea-architect` |
| پروژه تعریف شده، باید فازبندی شود | `02-task-planner` |
| قبل از کدنویسی، ساختار باید ترسیم شود | `03-architecture-mapper` |
| هنگام نوشتن هر خط کد | `04-clean-code` |
| باگ / خطا / رفتار غیرمنتظره / گیر کردن | `05-debug-protocol` |
| بعد از هر تغییر کد | `06-git-discipline` |
| قبل از اعلام «تمام شد» برای هر فاز | `07-qa-testing` |
| پایان هر جلسه یا هر تصمیم مهم | `01-project-memory` → ثبت |

## 🔄 جریان استاندارد یک پروژه (Golden Path)

```
شروع جلسه
   │
   ▼
[1] بازیابی حافظه (01) ─── پروژه‌ی جدید است؟ ──► [2] کشف ایده (08)
   │ ادامه‌ی کار قبلی                                    │
   ▼                                                    ▼
[3] پلن و فازبندی (02) ◄────────────────────────────────┘
   │
   ▼
[4] ترسیم معماری (03)  ← قبل از هر خط کد
   │
   ▼
[5] حلقه‌ی اجرا:
    ┌─────────────────────────────────────┐
    │ کدنویسی با استاندارد (04)           │
    │   ├─ گیر کردی؟ → پروتکل دیباگ (05) │
    │   ├─ تغییر دادی؟ → کامیت (06)      │
    │   └─ تسک تمام شد؟ → تست (07)       │
    └─────────────────────────────────────┘
   │
   ▼
[6] ثبت در حافظه (01): worklog + checkpoint + تصمیم‌ها
   │
   ▼
پایان جلسه → به‌روزرسانی state.md
```

## ⛔ قوانین آهنین (هرگز نقض نشود)

1. **هیچ کدی قبل از پلن نوشته نمی‌شود.** اگر پلن نیست → اول `02-task-planner`.
2. **هیچ جلسه‌ای بدون خواندن حافظه شروع نمی‌شود.** حتی اگر مطمئنی یادت هست.
3. **هیچ جلسه‌ای بدون به‌روزرسانی حافظه تمام نمی‌شود.**
4. **بیش از ۳ تلاش ناموفق روی یک مشکل = توقف اجباری** → برو `05-debug-protocol`. حدس‌زدنِ پشت‌سرهم ممنوع.
5. **هر تغییر کد = یک کامیت.** بدون استثنا (`06-git-discipline`).
6. **«تمام شد» فقط بعد از عبور از چک‌لیست `07-qa-testing` گفته می‌شود.**
7. **از محدوده‌ی فاز فعلی خارج نشو.** ایده‌ی جدید وسط کار؟ در `ideas-parking.md` ثبت کن و ادامه بده (ضد over-engineering).
